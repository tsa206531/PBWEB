# ADR-004: v3.0 範圍重評估（對齊 ADR-002/003；強型別 UI 延後、安全基礎前置）

## 狀態

提議中 (Proposed) - 2026-06-01

> 本 ADR 是對 [v3.0 PRD](../../L1/v3.0_PRD_StronglyTyped.md) 的**範圍重評估**，依 [ADR-002](ADR-002-商業化與部署模式.md)
> 與 [ADR-003](ADR-003-soul.md) 的結論調整 v3.0 的取捨。屬決策紀錄，尚未實作。

## 背景

- v3.0 PRD（2026-05-31）成文於 ADR-003 靈魂重定位（2026-06-01）**之前**，需對齊。
- **觸發點一**：強型別的**型別設定 UI（PRD §6.3）尚未定案**。
- **觸發點二**：ADR-003 將「把機密鎖好不外洩」定為**產品靈魂**，但 v3.0 PRD **未含加密與 token scoping**
  （現況：明文 SQLite `electron/db.cjs`、單一共用 token `electron/tokenStore.cjs`）——靈魂與實作相矛盾。
- **已定案（創辦人）**：
  1. 強型別「**只延後型別設定 UI**」，MCP Tools 改用**弱型別自動推導**本版照常上；
  2. **加密 + MCP token scoping 納入本版**。

## 決策總綱

重新切 v3.0 範圍為「**本版做 / 延後（後續分屬 v3.1、v3.2）**」，並把 ADR-003 的**安全基礎前置**進本版。

> 結論：**v3.0 PRD 十項中只有一項（§6.3 強型別 GUI）延後；其餘照做，並額外前置兩項安全工程（加密、token scoping）。**

| # | v3.0 PRD 項目 | 本版決定 | 依據 |
| :-- | :-- | :-- | :-- |

| 3 | 弱型別自動推導 | ✅ 本版做（後端，**零新 UI**） | PRD §6.2 |

### 二、本版新增（v3.0 PRD 未含、依 ADR-002/003 前置）

| 項目 | 內容 | 依據 |
| :-- | :-- | :-- |
| **靜態加密（SQLCipher）** | SQLite 改用加密版（`better-sqlite3-multiple-ciphers` 或等價）；金鑰以 `electron.safeStorage` 管理 | ADR-002 §7、ADR-003 §2 |
| **MCP token scoping** | 單一共用 token → 具名／scoped token + rate limit；驗證中介層強化 | ADR-002 §7、ADR-003 §2 |
| 機密暴露守則 | 本版以 binary 資料鎖落地「機密不進 MCP」；`is_locked` 卡片一律不出現在 Tools／Resources | ADR-003 §3 |

### 三、延後（後續再評估，分屬 v3.1 / v3.2）

**延後 v3.2（強型別）：**
- 強型別**型別設定 GUI**（§6.3）：toggle、per-variable 型別、enum／boolean／array、JSON Schema 預覽。
- 寫入 `schema_definition` 的 UI 路徑 + Ajv **強**驗證（非 string 型別）。

**延後 v3.1（機密深模型 + 串聯）：**
- ADR-003 §3 的**機密深模型**：機密分級、預設不自動吐出、釋放 consent、釋放 audit（本版先用 binary 鎖代替）。
- Prompt Chaining（PRD §9 方向 A，本就留下一版）。

### 五、與 ADR-003 靈魂的對齊檢查

- ✅ **資料鎖 + 靜態加密 + token scoping** = 兌現「鎖好不外洩」的核心三件。
- ⚠️ 仍缺（v3.1）：機密**釋放 consent／分級／audit**；「把密鑰餵 AI」的注入外洩風險，本版以
  「機密一律 `is_locked`、不進 Tools」為臨時護欄，完整威脅模型留 v3.1。

### 六、後果 / 風險

- **加密換原生模組**：跨平台需重新 rebuild；**既有明文 DB 的一次性加密遷移**必須處理
  （PRD 原本「不遷移」策略只適用於新增欄位；加密是新風險）→ 需決議：首次啟動偵測明文→就地加密 vs 引導重建。
- **本版變重**：納入安全工程後時程拉長（符合決議二的選擇）；但這是靈魂的及格證，值得。
- **仍無自動化測試**：加密、Tools、驗證、token scoping 都該補測試（呼應 ADR-002 §6「測試為前置」）。
- **強型別延後不破相容**：MCP Tools 先以弱型別上，未來補強型別不影響已連線 IDE（PRD 設計本就相容）。
