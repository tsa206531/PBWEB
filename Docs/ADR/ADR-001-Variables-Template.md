# ADR-001: 變數樣板功能設計 (Variables Template)

## 狀態

提議中 (Proposed) - 2026-02-15

## 背景

使用者希望在 PromptBox 中支援變數樣板功能，允許在 Prompt 內容中定義變數（如 `{{topic}}`），並在使用時動態填入值。此功能旨在提升 Prompt 的重用性與靈活性。

## 決策

### 1. UI 佈局位置

**決策**：Variables Panel 放置在 **PromptModal 的右側**（與 Modal 同層），而非內嵌在 Content 編輯區內。

**理由**：

- 未來計劃新增「串聯模板」功能，也會使用右側面板
- 統一的側邊欄模式可提供一致的使用體驗
- 可以同時查看完整的 Content 與 Variables

**布局示意**：

```
+--------------------------------+  +------------------+
| PromptModal                    |  | Variables Panel  |
|                                |  |                  |
| Title: [______________]        |  | {{topic}}        |
| Folder: [_______]              |  | = "__________"   |
|                                |  |                  |
| [編輯] [預覽] [編輯變數 🔧]      |  | {{style}}        |
|                                |  | = "__________"   |
| Content:                       |  |                  |
| [____________________]         |  |                  |
+--------------------------------+  +------------------+
```

---

### 2. 變數來源與格式

**決策**：系統自動掃描 Content 中的 `{{variable}}` 格式。

**規則**：

- **變數名稱格式**：`^[a-zA-Z_][a-zA-Z0-9_]*$`（英文字母、數字、底線，不可數字開頭）
- **不支援巢狀變數**：`{{user.name}}` 不會被識別
- **不合法的變數**：保留原樣，不加入 Variables Panel

**範例**：

- ✅ `{{topic}}`, `{{my_style}}`, `{{_temp}}`
- ❌ `{{topic-name}}`, `{{123abc}}`, `{{🚀}}`

---

### 3. 使用者填值方式

**決策**：純字串替換模式，無型別限制。

**規則**：

- 左側：系統自動生成變數名稱
- 右側：使用者填入字串值（可為空）
- 格式：`{{variable}} = "使用者填入的值"`
- 無需型別宣告（全部視為字串）

---

### 4. 變數預設值儲存

**決策**：將變數值儲存到資料庫的 `prompts` 表。

**理由**：

- 使用者可設定「常用預設值」
- 提升重複使用 Prompt 的效率
- 下次開啟時自動載入上次填入的值

**實作**：

- 新增欄位：`variables TEXT DEFAULT '{}'`（JSON 格式）
- 範例：`{"topic": "Vue 3", "style": "技術文章"}`

---

### 5. UI 互動行為

**決策**：「編輯變數」按鈕採用 **Toggle 模式**。

**行為**：

- 點擊「編輯變數」→ 右側 Variables Panel 展開
- 再次點擊 → Panel 收合
- 允許同時查看 Content 與 Variables

**替代方案（被拒絕）**：

- ❌ 分頁模式（如 Edit/Preview）：會隱藏 Content，不利於對照編輯

---

### 6. 複製到剪貼簿的行為

**決策**：依「變數功能開關」狀態決定行為。

| 功能開關 | 行為                               |
| :------- | :--------------------------------- |
| **開啟** | 使用預設值替換變數，複製最終內容   |
| **關閉** | 保留原始樣板（`{{topic}}` 不替換） |

**未填值的處理**：

- 若變數無預設值（空字串），則 **保留原樣** `{{topic}}`
- 範例：`{{topic}} = ""` → 複製時仍為 `{{topic}}`

---

### 7. Preview 模式顯示

**決策**：顯示 **替換後的內容**（使用預設值）。

**理由**：

- 符合「預覽」的語意（查看最終效果）
- 幫助使用者確認變數是否正確填入

**範例**：

- Content: `請分析 {{topic}} 的優缺點`
- Variables: `topic = "Pinia"`
- Preview: `請分析 Pinia 的優缺點`

---

### 8. 變數即時同步

**決策**：使用 **debounce (500ms)** 延遲掃描。

**理由**：

- 避免打字時頻繁重新掃描造成效能問題
- 使用者停止輸入 500ms 後才更新 Variables Panel

---

### 9. 重複變數處理

**決策**：同一變數名稱只顯示一次，填值後全部替換。

**範例**：

```
我喜歡 {{topic}}，{{topic}} 很有趣。
Variables Panel:
{{topic}} = "Vue 3"

替換後:
我喜歡 Vue 3，Vue 3 很有趣。
```

---

### 10. 卡片/列表模式的顯示

**決策**：依功能開關決定顯示方式。

| 功能開關 | 顯示內容                         |
| :------- | :------------------------------- |
| **開啟** | 顯示替換後的內容（使用預設值）   |
| **關閉** | 顯示原始樣板（保留 `{{topic}}`） |

---

## 後果

### 正面影響

- ✅ 提升 Prompt 重用性（一個樣板，多種情境）
- ✅ 降低重複編輯的成本
- ✅ 為未來「串聯模板」功能鋪路

### 負面影響

- ⚠️ 增加 UI 複雜度（需要 Toggle 與右側面板）
- ⚠️ 資料庫需要 Migration（新增 `variables` 欄位）
- ⚠️ 變數掃描與替換需要額外的效能成本

### 風險

- **變數名稱衝突**：若使用者不小心使用相同變數名，可能造成非預期替換
- **效能問題**：大型 Prompt 的即時掃描可能造成延遲（已透過 debounce 緩解）
